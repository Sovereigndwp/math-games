# Data Schema: Misconception Library

**Taskade Project ID:** `cyt3zvpjf32D1Ddt`  
**View:** List  
**Purpose:** A growing library of learner misconception patterns, organized by game and error category. Used by the Misconception Architect workflow and the Pipeline Orchestrator agent as knowledge.

---

## Entry Format

Every entry in the library follows this exact structure:

```
---
MISCONCEPTION: [plain-language name of the misconception]
CATEGORY: [see categories below]
DESCRIPTION: [What exactly does the player do wrong? Specific operation or reasoning pattern.]
FAILURE SIGNAL: [Observable behavior in the game that reveals this misconception — what would a designer or researcher see?]
DESIGN RESPONSE: [Specific mechanic, feedback, or constraint that directly addresses this misconception]
APPLIES TO FUTURE GAMES WHEN: [Conditions under which a future game would make this entry relevant]
LIBRARY ACTION: [KEEP | REVISE | NEW]
SEVERITY: [HIGH | MEDIUM | LOW]
---
```

New entries generated by the Misconception Architect workflow are written as `[PENDING REVIEW]` and must be manually promoted to active status.

---

## Misconception Categories

| Category | Meaning |
|---|---|
| `procedural_error` | Player applies the right procedure incorrectly (e.g. forgets to carry, wrong order of operations) |
| `representation_mismatch` | Player confuses one representation for another (e.g. fraction vs decimal, degrees vs radians) |
| `conceptual_gap` | Player lacks the underlying concept entirely (e.g. doesn't understand what division means operationally) |
| `overgeneralization` | Player extends a rule beyond where it applies (e.g. "multiplication always makes a number bigger") |
| `shortcut_skill` | Player finds a workaround that bypasses the target skill (e.g. trial-and-error instead of algebra) |
| `cognitive_overload` | Player is tracking too many things at once and abandons mathematical reasoning |

---

## Severity Ratings

| Severity | Meaning |
|---|---|
| `HIGH` | Will cause most players in the target age band to fail this type of question repeatedly |
| `MEDIUM` | Affects a significant minority; worth designing around |
| `LOW` | Edge case; affects few players; acceptable to leave unaddressed |

---

## Library Action Tags

| Action | Meaning |
|---|---|
| `KEEP` | Seeded entry transfers unchanged to this new game's misconception space |
| `REVISE` | Entry exists but needs rewriting because this game's brief changes the confusion risk |
| `NEW` | No existing library entry — must be generated fresh |
